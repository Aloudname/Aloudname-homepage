# game.md: 引力牧羊人 (Gravity Shepherd) — 彩蛋游戏设计文档

> **设计理念**: 利用引力物理 + 控制论原理，打造一个轻量、耐玩、有深度的隐藏小游戏
> **触发方式**: About 页顶部连续上滚（阈值 420px 等效，需刻意操作）
> **控制论映射**: 玩家 = PID 控制器，鼠标引力 = 控制信号，粒子 = 被控对象，捕获环 = 参考信号

---

## 1. 游戏概念

```
┌──────────────────────────────────────────────────┐
│                                                  │
│   ·  ·     ⭕（捕获环，Lissajous 运动）             │
│        ·        ·    ·                           │
│     ·      🧲（鼠标引力源）       ·               │
│  ·        ·        ·        ·                    │
│            ·    ·    ·                           │
│   ·              ·         ·                     │
│                                                  │
│   分数: 12    稳定性: ████████░░ 80%              │
│                           ⌄ 向下滚动返回          │
└──────────────────────────────────────────────────┘
```

**目标**: 用鼠标引力将粒子"甩"入不断运动的捕获环中。

**控制论本质**: 你是一个人类 PID 控制器——观察粒子与环之间的位置误差，用引力（比例项）修正轨迹，凭直觉完成轨迹规划（前馈项）。

## 2. 核心机制

### 2.1 粒子物理

| 属性 | 规则 |
|------|------|
| 边界 | 完全弹性碰撞（`v *= -1`，位置钳制到边界内） |
| 阻尼 | **无**（绝对光滑空间，粒子永动） |
| 鼠标引力 | `F = G * M_cursor * m_particle / r²`，方向指向鼠标 |
| 最大数量 | 24 个（轻量，保持帧率） |
| 初始速度 | 正态分布 N(avgSpeed, avgSpeed*0.3)，avgSpeed 随分数对数增长 |

### 2.2 捕获环（目标）

| 属性 | 规则 |
|------|------|
| 运动轨迹 | Lissajous 曲线: `x = W/2 + A*sin(a*t + δ)`, `y = H/2 + B*cos(b*t)` |
| 频率比 | a:b = 3:2（非整数比 → 轨迹不重复，不可预测） |
| 运动速度 | A, B 幅度随分数对数增长，`A = 0.25*W + log(1+score)*20`（上限 0.4*W） |
| 外观 | 发光虚线圆环，脉冲光晕，半径 30px |
| 捕获判定 | 粒子中心进入环内（`dist < ringRadius + particleRadius`） |

### 2.3 计分与失败

| 事件 | 效果 |
|------|------|
| 粒子被环捕获 | +1 分，粒子消失，环闪亮，+1 飘字 |
| 粒子飞出屏幕 | 稳定性 -5%，新粒子从边缘补充 |
| 稳定性 ≤ 0 | 游戏结束提示，3 秒后自动重置（分数归零） |
| 连续捕获 | 连击计数器，3 连击以上显示 combo 文字 |

### 2.4 难度曲线（对数增长 + 上限）

| 参数 | 公式 | Score=0 | Score=10 | Score=50 | 上限 |
|------|------|---------|----------|----------|------|
| avgSpeed | `min(2 + log(1+s)*1.2, 10)` | 2.0 | 4.9 | 7.0 | 10 |
| spawnRate | `min(1.5 + log(1+s)*0.5, 5)` | 1.5/s | 2.7/s | 3.5/s | 5/s |
| ringAmplitude | `0.25 + log(1+s)*0.04`（占屏幕比例） | 0.25 | 0.35 | 0.41 | 0.45 |
| ringSpeed | `min(0.6 + log(1+s)*0.4, 3)` (rad/s) | 0.6 | 1.6 | 2.2 | 3 |
| 粒子最大数 | 固定 | 24 | 24 | 24 | 24 |

## 3. 控制论分析

### 3.1 人机控制系统模型

```
                    ┌─────────────┐
  参考信号 r(t) ──→│  玩家(控制器) │──→ u(t) 引力 ──→│  粒子(被控对象) │──→ y(t) 位置
  (环位置)      ↑  │  (大脑+手)   │                  │ (质量+惯性)    │
                │  └─────────────┘                  └───────────────┘
                │                                          │
                └──────────── 视觉反馈 e(t)=r(t)-y(t) ─────┘
```

- **r(t)** = 捕获环位置（参考信号，Lissajous 运动）
- **y(t)** = 粒子位置（系统输出）
- **e(t)** = 位置误差（视觉反馈）
- **u(t)** = 引力矢量（控制信号，`F ∝ 1/r²`）
- **玩家** = PI 控制器：比例项（靠近环时用力拉）+ 积分项（累积误差修正）

### 3.2 为什么这个游戏"有深度"

1. **非线性动力学**: 引力是 `1/r²` 的，近距离控制极灵敏，远距离几乎无作用——玩家需要学习非线性增益调度
2. **非最小相位**: 粒子有惯性，控制力通过加速度间接影响位置——存在相位延迟，玩家必须"提前量"操作（前馈控制）
3. **时变参考信号**: 环不是静止的，玩家要同时预测环的轨迹和粒子的响应
4. **多目标权衡**: 多个粒子同时在场，玩家需要在各个粒子之间分配注意力（多变量控制）
5. **稳定性约束**: 不能无限失误——稳定性条倒逼玩家保守操作，形成风险管理的元游戏

## 4. 视觉效果

### 4.1 粒子渲染

- 颜色: 速度越快越亮（`alpha = 0.3 + speed/avgSpeed * 0.5`）
- 拖尾: 保留最近 6 帧位置，透明度递减
- 被捕获时: 6 方向爆发粒子 + 缩放消失

### 4.2 捕获环渲染

- 虚线圆环，外发光 `box-shadow: 0 0 20px rgba(0,200,255,0.4)`
- 脉冲动画: 大小呼吸（`scale 1.0 ↔ 1.08`，周期 1.5s）
- 捕获瞬间: 亮白色闪光 + 缩放弹跳

### 4.3 HUD

- 分数: 右上角大字体，捕获时 scale 弹跳
- 稳定性条: 底部居中，渐变色（绿→黄→红），剩余百分比
- Combo 指示: 3+ 连击时显示 "🔥 xN COMBO"

## 5. 技术架构

### 5.1 组件结构

```
src/views/public/
  └── GravityShepherd.vue     ← 新游戏组件（替代 ParticleGame.vue）

src/views/FirstView/
  └── FirstView.vue            ← 修改: 游戏组件引用改为 GravityShepherd
```

### 5.2 GravityShepherd.vue 内部结构

```javascript
// 核心数据
data: {
  score: 0,
  stability: 100,        // 0-100, 归零则游戏结束
  combo: 0,
  particles: [],          // 最多 24 个
  ring: { x, y, phase },
  mouse: { x, y, active },

  // 物理参数（对数曲线）
  avgSpeed: computed,
  ringAmplitude: computed,
  ringSpeed: computed,
  spawnRate: computed,
}

// 主循环
loop(ts):
  1. 生成/补充粒子（spawnRate, max 24）
  2. 更新环位置（Lissajous）
  3. 对每个粒子:
     a. 计算鼠标引力
     b. 更新位置（无阻尼）
     c. 弹性边界碰撞
     d. 检测环捕获
     e. 检测逃逸（飞出屏幕）
  4. 更新稳定性条
  5. 更新 combo 计时器
  6. 渲染（Canvas 2D）
```

### 5.3 与旧 ParticleGame 的区别

| 特性 | 旧 ParticleGame | 新 GravityShepherd |
|------|----------------|-------------------|
| 容器 | 随机游走容器 | Lissajous 环形捕获环 |
| 失败条件 | 无 | 稳定性条归零→游戏结束 |
| 粒子上限 | 32 | 24（更轻量） |
| 视觉效果 | 基础 | 环脉冲+连击+闪光+稳定性渐变色 |
| 退出方式 | 同（滚轮+底部⌄） | 同 |
| 游戏深度 | 低（纯机械） | 高（控制论深度+技能曲线） |

### 5.4 性能

- 24 粒子 × 每帧 O(n) 物理更新 = ~0.5ms/frame
- Canvas 2D 渲染 ~2ms/frame
- 总帧预算 < 5ms（60fps 预算 16.67ms 的 30%）
- v-if 懒加载 + 销毁时完全清理
